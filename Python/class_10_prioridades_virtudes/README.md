# Las Virtudes del Software

---------

Este módulo explora los principios fundamentales que definen un buen software.  Priorizar estas virtudes es crucial para desarrollar sistemas robustos, mantenibles y confiables. Nos inspiraremos en las ideas de Donald Knuth para establecer un orden lógico en estas prioridades.

## Principios Fundamentales

El buen código se caracteriza por claridad, corrección, eficiencia, flexibilidad y seguridad.  En este módulo, nos enfocaremos en las tres primeras virtudes como base para un desarrollo sólido.

### Claridad

La claridad del código es primordial. Un código claro es fácil de entender, modificar y mantener.  Esto se logra siguiendo los principios básicos de legibilidad y utilizando nombres descriptivos para variables y funciones.

### Corrección

La corrección es la virtud más importante. El software debe funcionar como se espera, produciendo los resultados correctos en todas las condiciones válidas.  Priorizar la corrección sobre la eficiencia es esencial para evitar errores y asegurar la confiabilidad del sistema.

### Eficiencia

Una vez garantizada la claridad y la corrección, la eficiencia se convierte en una consideración importante.  Optimizar el código para mejorar su rendimiento solo debe realizarse después de asegurar que cumple con los requisitos funcionales y es fácil de entender.

La frase clave de Donald Knuth, “la optimización prematura es la raíz de todos los males”, resalta la importancia de enfocarse primero en un código correcto y legible antes de preocuparse por su eficiencia.
